# 代码验收包

## 验收目标

确认系统能稳定生成、发布、归档和通知“YQN 每日重点简报”，并且不泄露密钥。

## 页面验收

- `/` 显示每日重点简报。
- `/archive/` 可查看历史简报。
- `/setup/` 可查看自动化配置状态。
- `/about/` 可查看系统说明。
- `/boss/`、`/executive/` 旧链接不 404，并提示已归入每日重点简报。

## 交互验收

- `复制简报摘要` 可用，提示 `已复制简报摘要`。
- `复制行动要点` 可用，提示 `已复制行动要点`。
- 搜索可用。
- 主题筛选可用。
- 上一天 / 下一天导航可用。
- 复制分享链接可用。
- 打印 / 保存 PDF 可用。
- 手机端无横向滚动。

## 生成验收

- `OPENAI_MODEL` 缺失时给清晰 setup error。
- `OPENAI_API_KEY` 缺失时只发布低信号配置提示，不伪装真实简报。
- Demo 样例明确标注。
- 每日最多输出 3 条重点。
- 每日生成仍按 09:37 Asia/Taipei。
- 飞书失败不阻断 Pages 部署。

## 飞书验收

成功卡片包含：

- `YQN 每日重点简报 · YYYY-MM-DD`
- 今日一句话判断
- 3 条重点预览
- `查看今日简报`
- `查看历史简报`

失败卡片包含：

- `每日简报生成失败 · YYYY-MM-DD`
- 失败阶段
- GitHub Actions run 链接
- 下一步建议

## 安全验收

- 不泄露 `OPENAI_API_KEY`。
- 不泄露 `FEISHU_WEBHOOK_URL`。
- 不泄露 `PAGE_ACCESS_PASSPHRASE`。
- noindex 和 robots.txt 保留。
- 加密开启时，完整日报和 search-index 不以明文出现在 dist。
- 文档明确说明 GitHub Pages 不是企业级登录系统。
