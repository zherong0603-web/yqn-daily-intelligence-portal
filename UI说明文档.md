# UI 说明文档

## 产品定位

YQN 每日重点简报是一个轻量情报门户。默认给用户本人每天 3 分钟读完，也能体面转发给团队或管理层。

它不是增长作战台，不是线索 KPI 看板，也不是工程 dashboard。首页只回答三件事：

1. 今天一句话判断是什么。
2. 今天 3 个重点是什么。
3. 今天建议做什么。

## 主导航

主导航只保留四个入口：

1. 今日简报
2. 历史简报
3. 自动化配置
4. 系统说明

旧 `/boss/`、`/executive/` 链接保留兼容，但不进入主导航。

## 首页结构

首页首屏显示：

1. `YQN 每日重点简报`
2. 今日一句话判断
3. 当前模式、OpenAI、飞书、加密状态
4. Demo 样例提示
5. `复制简报摘要`
6. `查看今日建议动作`
7. `查看历史简报`
8. `完成自动化配置`

## 每条重点结构

每个重点固定展示：

1. 发生了什么
2. 为什么重要
3. 对 YQN 的启发
4. 建议动作
5. 来源

如果当天没有强信号，页面显示“今日无强信号”，不强行凑机会。

## 搜索和历史

首页保留搜索框和主题筛选。历史页支持按日期、按月、按周查看。

## 自动化配置页

`/setup/` 只做配置和测试：

1. OpenAI API Key
2. OPENAI_MODEL
3. 飞书 webhook
4. 加密密码
5. 手动生成一次

页面只显示“已配置 / 缺失”，不显示任何 secret 值。

## 系统说明页

`/about/` 用于 OPC showcase，只说明这是“自动生成每日重点简报的轻量情报门户”，并写清楚安全边界和成本边界。

## 移动端

手机端优先显示今日判断、3 个重点、建议动作和搜索入口。页面不得横向滚动，按钮文字必须完整显示。
