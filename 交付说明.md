# 交付说明

2. 点阅读全文。
3. 看管理层摘要。
4. 看今日行动。
5. 查历史或分享给老板。

## 页面分别是干什么的

- 首页：YQN Growth War Room，当天最重要的入口，先看今日判断、MQL、组织缺口、内容实验和历史入口。
- 管理层摘要：老板 30 秒看懂今天，包含一句话判断、MQL 质量、风险、跨部门动作和今日 3 件事。
- MQL 质量：看线索数量、质量判断、转化判断、客户画像完整率、有效/无效原因。
- 组织缺口：看数据、销售反馈、私域承接、素材产能缺口，以及需要谁配合。
- 内容实验：看实景视频、痛点选题、销售 SOP 转内容、小红书/直播/短视频实验。
- 个人日报：复制到自己的日报或周报，说明今天做了什么、发现了什么、明天做什么、需要谁支持。
- 历史归档：按日期、按月、按周回看历史日报。
- 报告页：看某一天完整日报，支持上一天/下一天、复制分享链接、复制老板摘要、复制行动清单、打印 PDF。

## 历史怎么查

- 按日期：进入归档页，点击某一天日报。
- 按月：点击类似 `2026/07 月` 的入口。
- 按周：点击类似 `2026/week-27` 的入口。
- 搜索：在首页搜索框输入“美国仓”“小红书”“AI”等关键词。
- 主题筛选：在首页搜索区选择主题，只看某一类情报。

## 怎么分享给老板

- 发今日链接：打开今日日报页，点“复制分享链接”。
- 发管理层摘要：打开 `/executive/`，点“复制老板版摘要”。
- 复制老板版摘要：首页和管理层摘要页都有按钮。
- 打印 PDF：打开报告页，点“打印 / 保存 PDF”，在系统打印窗口选择保存为 PDF。

## 还缺哪些配置

### 1. OPENAI_API_KEY

- 打开哪里：GitHub 仓库 → Settings → Secrets and variables → Actions → Secrets。
- 点哪里：New repository secret。
- Name 填什么：`OPENAI_API_KEY`。
- Value 填什么：你的 OpenAI API Key。
- 保存按钮叫什么：Add secret。
- 配错会怎样：真实 AI 日报无法生成，Actions 会报错或只生成配置提示。
- 为什么不能由 Codex 代填：这是你的 OpenAI 私密密钥，不能发到聊天里，也不能写进仓库。

### 2. OPENAI_MODEL

- 打开哪里：GitHub 仓库 → Settings → Secrets and variables → Actions → Variables。
- 点哪里：New repository variable。
- Name 填什么：`OPENAI_MODEL`。
- Value 填什么：你 OpenAI API 账号里可用的模型名，优先低成本 mini 模型。
- 保存按钮叫什么：Add variable。
- 配错会怎样：Actions 会提示模型不可用或配置错误。
- 为什么不能由 Codex 代填：Codex 不能确认你账号实际可用的模型列表。

### 3. FEISHU_WEBHOOK_URL

- 打开哪里：GitHub 仓库 → Settings → Secrets and variables → Actions → Secrets。
- 点哪里：New repository secret。
- Name 填什么：`FEISHU_WEBHOOK_URL`。
- Value 填什么：飞书自定义机器人的 webhook 地址。
- 保存按钮叫什么：Add secret。
- 配错会怎样：网页仍部署，但飞书收不到通知。
- 为什么不能由 Codex 代填：这是飞书机器人私密推送地址，不能发到聊天里，也不能写进仓库。

### 4. PAGE_ACCESS_PASSPHRASE

- 打开哪里：GitHub 仓库 → Settings → Secrets and variables → Actions → Secrets。
- 点哪里：New repository secret。
- Name 填什么：`PAGE_ACCESS_PASSPHRASE`。
- Value 填什么：你自己设定的页面访问密码。
- 保存按钮叫什么：Add secret。
- 配错会怎样：开启加密后构建失败，或页面无法解锁正文。
- 为什么不能由 Codex 代填：这是访问密码，必须由你自己设置和保管。

## 成本边界

- ChatGPT Pro 不等于 API 免费额度。
- 自动日报必须使用 OpenAI API Key。
- API 按 OpenAI 官方计费规则单独计费。
- GitHub Pages / GitHub Actions 当前方案尽量免费。
- 搜索增强可能额外收费。

## 安全边界

- GitHub Pages 是公开网页。
- noindex/robots 不是访问控制。
- 加密模式是客户端加密，不是企业级登录。
- 不要放客户名单、报价、合同、内部成本、密钥、飞书 webhook、页面密码、私密线索。
