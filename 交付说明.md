# 交付说明

## 一句话结论

YQN Daily Intelligence Portal 已经可以作为每日情报门户使用：能打开首页、查看日报、查历史、搜索、按月/按周回看、复制分享链接、打印或保存 PDF。真实 AI 日报和飞书通知还需要你在 GitHub 填入自己的 OpenAI API Key、模型名和飞书机器人地址。

正式访问地址：https://zherong0603-web.github.io/yqn-daily-intelligence-portal/

## 每天什么时候发生

- 09:37 Asia/Taipei 开始生成。
- 09:45-10:05 Asia/Taipei 目标飞书通知。
- 成功时：飞书收到入口卡片，卡片里有今日判断、3 条核心预览、今日网页按钮和历史归档按钮。
- 失败时：飞书收到失败卡片，卡片里有失败阶段、GitHub Actions run 链接和下一步建议。
- 如果没有配置飞书机器人，网页仍会部署，只是不发飞书通知。

## 我每天怎么用

1. 打开飞书卡片。
2. 点“阅读全文”。
3. 先看今日一句话判断。
4. 再看今日核心信号。
5. 最后看今日行动清单。
6. 需要汇报时，复制链接或打印 / 保存 PDF。

## 历史怎么查

- 按日期：打开首页或归档页，点击某一天日报。
- 按月：进入 `/archive/YYYY/MM/`，例如 `/archive/2026/07/`。
- 按周：进入 `/archive/YYYY/week-WW/`，例如 `/archive/2026/week-27/`。
- 搜索：在首页搜索框输入“美国仓”“小红书”“AI”等关键词。
- 主题筛选：在首页搜索区选择主题，只看某一类信号。

## 怎么分享给老板

1. 发今日链接：打开今日日报页，点“复制分享链接”，直接发给老板。
2. 发某天历史链接：进入归档页，打开某天日报，点“复制分享链接”。
3. 打印成 PDF：打开日报页，点“打印 / 保存 PDF”，在系统打印窗口选择保存为 PDF。

报告页还有两个快捷按钮：

- “复制老板版摘要”：适合直接发给老板。
- “复制今日行动清单”：适合发给自己或团队跟进。

## 我还必须配置什么

### 1. OPENAI_API_KEY

- 打开哪里：GitHub 仓库 → Settings → Secrets and variables → Actions → Secrets。
- 点哪里：New repository secret。
- Name 填什么：`OPENAI_API_KEY`。
- Value 填什么：你的 OpenAI API Key。
- 保存按钮叫什么：Add secret。
- 配错会怎么样：真实 AI 日报无法生成，只能看到配置待完成或失败提示。
- 为什么不能由 Codex 代填：这是你账号的敏感密钥，不能发到聊天里，也不能写进仓库。

### 2. OPENAI_MODEL

- 打开哪里：GitHub 仓库 → Settings → Secrets and variables → Actions → Variables。
- 点哪里：New repository variable。
- Name 填什么：`OPENAI_MODEL`。
- Value 填什么：你 OpenAI API 账号里可用的模型名，优先选择低成本 mini 模型。
- 保存按钮叫什么：Add variable。
- 配错会怎么样：真实 AI 日报会在 Actions 里失败，并提示模型不可用或配置错误。
- 为什么不能由 Codex 代填：Codex 不能确认你 OpenAI API 账号实际可用的模型列表。

### 3. FEISHU_WEBHOOK_URL

- 打开哪里：GitHub 仓库 → Settings → Secrets and variables → Actions → Secrets。
- 点哪里：New repository secret。
- Name 填什么：`FEISHU_WEBHOOK_URL`。
- Value 填什么：飞书自定义机器人的 webhook 地址。
- 保存按钮叫什么：Add secret。
- 配错会怎么样：网页仍能部署，但飞书不会收到通知。
- 为什么不能由 Codex 代填：这是飞书机器人的私密推送地址，不能发到聊天里，也不能写进仓库。

### 4. PAGE_ACCESS_PASSPHRASE

- 打开哪里：GitHub 仓库 → Settings → Secrets and variables → Actions → Secrets。
- 点哪里：New repository secret。
- Name 填什么：`PAGE_ACCESS_PASSPHRASE`。
- Value 填什么：你自己设定的页面访问密码。
- 保存按钮叫什么：Add secret。
- 配错会怎么样：如果 `BRIEF_ENCRYPTION_ENABLED=true`，站点构建会失败，因为没有密码就不能安全发布加密内容。
- 为什么不能由 Codex 代填：这是访问密码，必须由你自己设置和保管。

## 安全边界

- GitHub Pages 是公开网页，任何知道链接的人都可能打开。
- `noindex` 和 `robots.txt` 只是不希望搜索引擎收录，不是访问控制。
- 加密模式会保护完整正文和搜索索引，但这是客户端加密，不是企业级登录系统。
- 不要放客户名单、报价、合同、内部成本、密钥、飞书 webhook、私密线索或未公开经营数据。
- 不要把任何密钥发到聊天里。

## 成本边界

- ChatGPT Pro 不等于 OpenAI API 免费额度。
- 自动日报必须使用 OpenAI API Key。
- API 按 token 单独计费，以 OpenAI 官方计费规则为准。
- GitHub Pages / GitHub Actions 在当前方案下尽量免费。
- 如果未来开启搜索增强，可能产生额外 API 或搜索服务费用。

## 常见故障和对应操作

### 页面能打开，但没有真实 AI 日报

检查 `OPENAI_API_KEY` 和 `OPENAI_MODEL` 是否已配置。配置完成后到 Actions 手动运行一次 Daily Briefing Portal。

### Actions 提示缺少 OPENAI_MODEL

到 GitHub 仓库 → Settings → Secrets and variables → Actions → Variables，添加 `OPENAI_MODEL`。

### Actions 提示 OpenAI API 失败

检查 `OPENAI_API_KEY` 是否正确、OpenAI 账号是否有 API 权限、模型名是否可用、账号是否有余额。

### 飞书没有收到通知

检查 `FEISHU_WEBHOOK_URL` 是否已配置。飞书失败不会阻断网页部署。

### 开启加密后页面打不开正文

确认 `BRIEF_ENCRYPTION_ENABLED=true` 时已经配置 `PAGE_ACCESS_PASSPHRASE`。如果输错密码，正文不会显示。

### 想立刻手动生成一次

打开 GitHub 仓库 → Actions → Daily Briefing Portal → Run workflow。真实日报时不要勾选 sample_backfill；验收样例时可以勾选 sample_backfill 并填写日期列表。
