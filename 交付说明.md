# 交付说明

## 当前项目是否已经上线

已上线 Demo 样例门户。正式真实 AI 简报需要用户本人在 GitHub 配置 OpenAI API Key 和 `OPENAI_MODEL` 后，手动测试一次。

正式访问地址：

`https://zherong0603-web.github.io/yqn-daily-intelligence-portal/`

## 每天什么时候生成和通知

- 每天生成：09:37 Asia/Taipei。
- 每天通知：目标 09:45-10:05 Asia/Taipei。
- 飞书只发送入口卡片，不发送完整正文。

## 历史怎么查看

- 按日期：打开首页或历史简报页，点击某一天。
- 按月：打开历史简报页，点击 `2026/07` 这类月份入口。
- 按周：打开历史简报页，点击 `2026/week-27` 这类周入口。
- 搜索：在首页搜索框输入“美国仓”“小红书”“AI”等关键词。
- 主题筛选：在搜索区域选择主题后查看结果。

## 如何分享

打开当天简报页，点击“复制分享链接”或“复制简报摘要”，再粘贴到飞书、微信或邮件。

## 如何打印或保存 PDF

打开当天简报页，点击“打印 / 保存 PDF”。系统会调用浏览器打印窗口，在打印窗口里选择保存为 PDF。

## 还缺哪些配置

- `OPENAI_API_KEY`：真实 AI 简报必须配置。
- `OPENAI_MODEL`：真实 AI 简报必须配置。
- `FEISHU_WEBHOOK_URL`：需要飞书通知时配置。
- `PAGE_ACCESS_PASSPHRASE`：开启加密模式时配置。

这些配置不能由 Codex 代填，因为它们是密钥、webhook 或访问密码，不能发到聊天里，也不能写进代码。

## 手动配置 GitHub Secrets

1. 打开 GitHub 仓库。
2. 点击 `Settings`。
3. 点击 `Secrets and variables`。
4. 点击 `Actions`。
5. 进入 `Secrets`。
6. 点击 `New repository secret`。
7. `Name` 填对应名称，例如 `OPENAI_API_KEY`。
8. `Value` 粘贴对应值。
9. 点击 `Add secret` 保存。

## 手动配置 GitHub Variables

1. 打开 GitHub 仓库。
2. 点击 `Settings`。
3. 点击 `Secrets and variables`。
4. 点击 `Actions`。
5. 进入 `Variables`。
6. 点击 `New repository variable`。
7. `Name` 填 `OPENAI_MODEL`。
8. `Value` 建议先填 `gpt-4o-mini`，这是当前更省钱的 OpenAI 小模型；如果账号不可用，再按 OpenAI 模型文档换成账号可用模型。
9. 点击 `Add variable` 保存。

## 手动开启 GitHub Pages

1. 打开 GitHub 仓库。
2. 点击 `Settings`。
3. 点击 `Pages`。
4. 在 `Source` 选择 `GitHub Actions`。
5. 点击保存。

## 手动触发日报

1. 打开 GitHub 仓库。
2. 点击 `Actions`。
3. 点击 `Daily Briefing Portal`。
4. 点击 `Run workflow`。
5. 真实简报不要勾选 sample/backfill。
6. 点击绿色的 `Run workflow`。
7. 等 5-10 分钟后打开首页查看。

## 安全边界

GitHub Pages 是公开网页。`noindex` 和 `robots.txt` 不是访问控制。

不要放客户名单、报价、合同、内部成本、密钥或私密线索。加密模式是客户端加密，不是企业级登录系统。

## 成本边界

ChatGPT Pro 不等于 OpenAI API 免费额度。自动日报必须使用 OpenAI API Key，API 按 OpenAI 官方计费规则单独计费。

## API 省钱选择

当前网站已经支持 OpenAI API。最少改动、最稳妥的省钱方案是：

1. `OPENAI_MODEL` 填 `gpt-4o-mini`。
2. 先不开启 `OPENAI_WEB_SEARCH_ENABLED`。
3. 先每天只跑一次，不做反复手动重跑。

如果要用免费额度路线：

1. Google Gemini API 有 Free Tier，但需要后续把网站接入 `GEMINI_API_KEY`。
2. 阿里云 Model Studio 新用户有免费额度，但只在指定区域和指定模型规则下生效，也需要后续接入对应 API。
3. 不建议把网页 cookie、模拟登录或 ChatGPT 网页会员拿来自动跑日报。

## 常见故障和对应操作

- 首页只有 Demo：配置 `OPENAI_API_KEY` 和 `OPENAI_MODEL` 后手动跑一次 Actions。
- Actions 提示模型不可用：检查 `OPENAI_MODEL` 是否是你账号可用的模型名。
- 网页更新但飞书没通知：检查 `FEISHU_WEBHOOK_URL` 是否已配置，飞书失败不会阻断网页。
- 加密页打不开正文：检查 `PAGE_ACCESS_PASSPHRASE` 是否配置，重新跑一次 Actions。
- 搜索没有结果：换关键词，或确认历史简报里是否有相关主题。
