# YQN Daily Intelligence Portal 交付说明

## 当前项目是否已经上线

项目已经部署到 GitHub Pages 发布通道。正式页面会由 GitHub Actions 自动发布。

正式访问地址：

https://zherong0603-web.github.io/yqn-daily-intelligence-portal/

GitHub 仓库：

https://github.com/zherong0603-web/yqn-daily-intelligence-portal

GitHub Actions：

https://github.com/zherong0603-web/yqn-daily-intelligence-portal/actions/workflows/daily-briefing.yml

## 每天什么时候生成

系统每天 `09:37 Asia/Taipei` 自动开始生成日报。

GitHub Actions 使用 UTC 时间，所以工作流里写的是 `01:37 UTC`，等于台北时间 `09:37`。

## 每天什么时候通知

目标是在 `09:45-10:05 Asia/Taipei` 完成网页部署并发送飞书通知。

飞书只发送入口卡片，不发送大段正文。

## 历史怎么查看

- 按日期：打开 `/reports/YYYY-MM-DD/`
- 全部归档：打开 `/archive/`
- 按月归档：打开 `/archive/YYYY/MM/`
- 按周归档：打开 `/archive/YYYY/week-WW/`
- 搜索：在首页搜索框输入关键词
- 主题筛选：在首页搜索区选择主题

## 如何分享给老板

打开某一天的日报页，点击“复制分享链接”，把复制出来的网页链接发给老板。

每一天都有永久链接，例如：

`/reports/2026-07-01/`

## 如何打印或保存 PDF

打开某一天的日报页，点击“打印 / 保存 PDF”。

浏览器会打开打印窗口，可以选择打印机，也可以选择“保存为 PDF”。

## 还缺哪些配置

真实每日自动生成前，用户必须在 GitHub 手动配置：

- `OPENAI_API_KEY`
- `OPENAI_MODEL`
- `FEISHU_WEBHOOK_URL`
- `PAGE_ACCESS_PASSPHRASE`，建议配置，开启加密模式时必填

可选配置：

- `FEISHU_SIGN_SECRET`
- `BRIEF_ENCRYPTION_ENABLED`
- `SITE_URL`
- `OPENAI_WEB_SEARCH_ENABLED`
- `MAX_SEARCH_CALLS`

## 哪些配置必须用户手动填，为什么不能由 Codex 代填

密钥属于账号级敏感信息，不能发在聊天里，也不能写进代码、README、网页、日志或附件。

Codex 只能实现读取 GitHub Secrets / Variables 的逻辑，不能替你知道或保存这些密钥。

## 手动配置 GitHub Secrets 的路径

1. 打开 GitHub 仓库：`https://github.com/zherong0603-web/yqn-daily-intelligence-portal`
2. 点击 `Settings`
3. 点击左侧 `Secrets and variables`
4. 点击 `Actions`
5. 点击 `New repository secret`
6. 填写 `Name`
7. 填写 `Secret`
8. 点击 `Add secret`

需要添加的 Secrets：

- Name: `OPENAI_API_KEY`，Value: 你的 OpenAI API Key
- Name: `FEISHU_WEBHOOK_URL`，Value: 飞书自定义机器人 webhook
- Name: `FEISHU_SIGN_SECRET`，Value: 飞书机器人签名密钥，没有开启签名可不填
- Name: `PAGE_ACCESS_PASSPHRASE`，Value: 页面访问密码，开启加密时必填

## 手动配置 GitHub Variables 的路径

1. 打开 GitHub 仓库
2. 点击 `Settings`
3. 点击左侧 `Secrets and variables`
4. 点击 `Actions`
5. 点击 `Variables`
6. 点击 `New repository variable`
7. 填写 `Name`
8. 填写 `Value`
9. 点击 `Add variable`

需要添加的 Variables：

- Name: `OPENAI_MODEL`，Value: 你要使用的 OpenAI API 模型名
- Name: `SITE_URL`，Value: `https://zherong0603-web.github.io/yqn-daily-intelligence-portal`
- Name: `BRIEF_ENCRYPTION_ENABLED`，Value: `true` 或 `false`
- Name: `OPENAI_WEB_SEARCH_ENABLED`，Value: `false`
- Name: `MAX_SEARCH_CALLS`，Value: `0`

## 手动开启 GitHub Pages 的路径

如果页面无法访问，按这个路径检查：

1. 打开 GitHub 仓库
2. 点击 `Settings`
3. 点击左侧 `Pages`
4. 找到 `Source`
5. 选择 `GitHub Actions`
6. 点击保存

当前仓库已尝试自动开启 GitHub Actions 发布模式。

## 手动触发日报的路径

1. 打开 GitHub 仓库
2. 点击 `Actions`
3. 点击左侧 `Daily Briefing Portal`
4. 点击右侧 `Run workflow`
5. 普通补跑：填写 `brief_date`，例如 `2026-07-01`
6. 验收样例补跑：勾选 `sample_backfill`
7. 在 `brief_dates` 填入：`2026-07-01,2026-07-02,2026-08-01`
8. 点击绿色 `Run workflow`

## 安全边界

GitHub Pages 是公开网页，不是企业级登录系统。

即使已经设置 `noindex` 和 `robots.txt`，也不要放：

- 客户名单
- 报价
- 合同
- 内部成本
- API Key
- 飞书 webhook
- 页面访问密码
- 私密线索或隐私数据

如果 `BRIEF_ENCRYPTION_ENABLED=true`，完整日报、历史正文和搜索索引会加密发布；浏览器输入密码后本地解密。

## 成本边界

ChatGPT Pro 不等于 OpenAI API 免费额度。

自动日报必须使用 `OPENAI_API_KEY`，API 调用会按 OpenAI 官方计费规则单独计费。

V1 优先使用免费能力：

- GitHub Pages
- GitHub Actions
- RSS / Atom
- 公开网页来源

OpenAI API 是唯一必要可变成本。

## 常见故障和对应操作

### 页面打不开

检查 GitHub Pages 是否选择 `GitHub Actions`，路径：`Settings → Pages → Source → GitHub Actions`。

### 每日没有正常生成

打开 `Actions → Daily Briefing Portal`，查看最近一次运行是否失败。

### 提示缺少 OPENAI_MODEL

到 `Settings → Secrets and variables → Actions → Variables` 添加 `OPENAI_MODEL`。

### 提示缺少 OPENAI_API_KEY

到 `Settings → Secrets and variables → Actions → Secrets` 添加 `OPENAI_API_KEY`。

### 飞书没有通知

检查 `FEISHU_WEBHOOK_URL` 是否已经配置。飞书失败不会阻断网页部署。

### 加密模式打不开全文

检查 `BRIEF_ENCRYPTION_ENABLED=true` 时是否已经配置 `PAGE_ACCESS_PASSPHRASE`。

### 搜索没有结果

先确认已有日报。没有历史日报时，搜索和日历会显示空状态。
