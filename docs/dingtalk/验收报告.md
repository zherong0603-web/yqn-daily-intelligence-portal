# 验收报告

## 验收范围

V1.3：Markdown 简洁版 + 真实公开来源采集 + demo/live 结构化生成 + 钉钉测试群 + GitHub Pages 归档验收。

## 已覆盖能力

- 工作日北京时间 08:45 定时触发。
- UTC cron 为 `45 0 * * 1-5`。
- 工作日北京时间 06:00 预抓取 dry-run。
- 预抓取 UTC cron 为 `0 22 * * 0-4`。
- 支持 `workflow_dispatch` 手动补跑。
- demo 模式不依赖 OpenAI API。
- live 模式先抓取真实公开 RSS / 网页候选源。
- live 模式支持 OpenAI API；没有 key 时优先使用 GitHub Models。
- 模型不可用时使用真实候选源 fallback，不回退 demo。
- 生成 `source_report.json`，记录候选源、分数原因和来源错误。
- 使用 zod 校验五模块日报结构。
- 群内消息控制为前三条高权重信号。
- 完整归档链接放在群消息开头。
- 群消息不展示动作清单。
- 不展示“是否敏感：否”。
- 不展示伪精确百分比置信度。
- 来源展示为“来源名称｜日期”。
- 生成 `dist/dingtalk` 归档页。
- 部署 GitHub Pages 后再做归档 HTTP 校验。
- 发送前生成 `archive_link_check.json`、`risk_report.json`、`validation_report.json`。
- 生成 Markdown 预览、HTML 预览、桌面截图、手机截图和归档截图。
- 命中 P0 风险会阻断发送。

## 进入正式群前检查

- 测试群连续 3 个工作日正常。
- 归档链接连续可打开。
- 内容无客户名单、报价、合同、毛利、内部成本和未公开案例。
- `DINGTALK_FORMAL_GROUP_ENABLED` 未确认前保持 `false`。
- 正式群 webhook 不与测试群混用。
