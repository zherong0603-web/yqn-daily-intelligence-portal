# 验收报告

## 验收范围

V1.4：北美三路强制联网搜索 + 2/2/1 配比 + 卖家价值评分 + 多群独立发送 + 钉钉预览与归档验收。

## 已覆盖能力

- 工作日北京时间 08:45 定时触发。
- UTC cron 为 `45 0 * * 1-5`。
- 工作日北京时间 06:00 预抓取 dry-run。
- 预抓取 UTC cron 为 `0 22 * * 0-4`。
- 支持 `workflow_dispatch` 手动补跑。
- demo 模式不依赖 OpenAI API。
- live 模式必须使用 OpenAI Responses API `web_search`；缺少 API key 或模型时停止发送。
- 美国、墨西哥和美墨联动分别联网，生成 `source_report.json` 留下完成状态与来源记录。
- 使用 zod 校验固定五条结构，并强制 2 条美国、2 条墨西哥、1 条联动。
- 群内消息只允许 70 分以上、官方来源且至少映射一个物流环节的信号。
- 完整归档链接放在群消息开头。
- 群消息不展示动作清单。
- 不展示“是否敏感：否”。
- 不展示伪精确百分比置信度。
- 来源展示为“来源名称｜日期”。
- 生成 `dist/dingtalk` 归档页。
- 部署 GitHub Pages 后再做归档 HTTP 校验。
- 发送前生成 `archive_link_check.json`、`risk_report.json`、`validation_report.json`。
- 生成 Markdown 预览、HTML 预览、桌面截图、手机截图和归档截图。
- 命中 P0 风险会阻断发送。
- 手工补充版未获得发送确认时会写入 `send_authorized=false`；非 dry run 会被阻断。
- 正式群和直播天团分别保存成功标记，同日重复运行只跳过已成功的目标。

## 进入正式群前检查

- 本地预览和测试群连续 3 个工作日正常。
- 归档链接连续可打开。
- 内容无客户名单、报价、合同、毛利、内部成本和未公开案例。
- `DINGTALK_FORMAL_GROUP_ENABLED` 未确认前保持 `false`。
- `DINGTALK_YQN_LIVE_GROUP_ENABLED` 未确认前保持 `false`。
- 正式群 webhook 不与测试群混用。
