# 配置说明

## 钉钉测试群

### DINGTALK_WEBHOOK_URL

- 是否必须：真实发送必须；dry run 可不填。
- 打开哪里：GitHub 仓库 Settings。
- 点哪里：Secrets and variables -> Actions -> New repository secret。
- Name：`DINGTALK_WEBHOOK_URL`
- Value：钉钉测试群自定义机器人 webhook。
- 保存按钮：Add secret。
- 配错后表现：发送步骤失败，或钉钉群收不到消息。

### DINGTALK_SECRET

- 是否必须：机器人启用加签时必须。
- 打开哪里：GitHub 仓库 Settings。
- 点哪里：Secrets and variables -> Actions -> New repository secret。
- Name：`DINGTALK_SECRET`
- Value：钉钉机器人加签 secret。
- 保存按钮：Add secret。
- 配错后表现：钉钉返回加签校验失败。

## YQN 直播天团

机器人在钉钉群内命名为 `YQN 信息小助手`。新增目标不会覆盖现有正式群。

- `DINGTALK_YQN_LIVE_GROUP_WEBHOOK_URL`：直播天团自定义机器人 webhook，保存为 GitHub Secret。
- `DINGTALK_YQN_LIVE_GROUP_SECRET`：机器人启用加签时的 secret，保存为 GitHub Secret。
- `DINGTALK_YQN_LIVE_GROUP_ENABLED`：是否启用直播天团发送，保存为 GitHub Variable；预览验收前保持 `false`，正式启用时改为 `true`。
- 正式群和直播天团分别记录发送标记；同一天已成功的群会单独跳过，不影响另一个群补发。

## Ryan 内部群（正式副本）

群内复用已有的 `YQN 北美履约增长晨报` 机器人，不新建 webhook，也不复制密钥。

- `DINGTALK_RYAN_INTERNAL_GROUP_ENABLED`：是否将 Seller Problem Brief 同步到 Ryan 内部群，保存为 GitHub Variable。
- 该目标复用现有 `DINGTALK_WEBHOOK_URL` / `DINGTALK_SECRET`，只在 Seller Problem Brief workflow 内映射为独立目标。
- 直播天团与 Ryan 内部群分别预检、分别记录发送尝试和成功标记；任一群失败不会阻止另一个群执行。
- 同一天补跑时，每个群独立去重；出现“已尝试但没有成功回执”的群会停止自动重试，避免重复消息。

## 归档链接

### PUBLIC_BASE_URL / PAGES_BASE_URL

- 是否必须：建议配置；归档链接验收必须能得到正确 Pages 根地址。
- 打开哪里：GitHub 仓库 Settings。
- 点哪里：Secrets and variables -> Actions -> Variables -> New repository variable。
- Name：`PUBLIC_BASE_URL` 或 `PAGES_BASE_URL`
- Value：`https://zherong0603-web.github.io/yqn-daily-intelligence-portal`
- 保存按钮：Add variable。
- 配错后表现：归档链接检查失败，群内显示“归档暂未启用”，不会发送 404 链接。

## live 模式

### OPENAI_API_KEY

- 是否必须：live 模式使用 OpenAI API 时必须；demo 模式不需要。
- 打开哪里：GitHub 仓库 Settings。
- 点哪里：Secrets and variables -> Actions -> New repository secret。
- Name：`OPENAI_API_KEY`
- Value：OpenAI API key。
- 保存按钮：Add secret。
- 配错后表现：live 生成失败或回退到测试样例。

### OPENAI_MODEL

- 是否必须：live 模式使用 OpenAI API 时必须；demo 模式不需要。
- 打开哪里：GitHub 仓库 Settings。
- 点哪里：Secrets and variables -> Actions -> New repository secret 或 Variables。
- Name：`OPENAI_MODEL`
- Value：账号可用且支持 Responses API `web_search` 的模型名；默认使用 OpenAI 官方快速入门示例中的 `gpt-5`。
- 保存按钮：Add secret 或 Add variable。
- 配错后表现：live 生成或强制联网搜索失败，发送被阻断。

### 强制联网搜索

- `OPENAI_WEB_SEARCH_ENABLED`：live 模式必须为 `true`，主工作流已强制设置。
- `MAX_SEARCH_CALLS`：必须至少为 `3`，分别完成美国、墨西哥和美墨联动搜索。
- 任一区域没有完成搜索、来源未被联网结果验证、2+2+1 数量不足，发送都会被阻断。

## 手动运行

1. 打开 GitHub 仓库 Actions。
2. 选择 `DingTalk YQN North America Intelligence V1.4`。
3. 点 Run workflow。
4. `mode` 先选 `live`。
5. `dry_run` 先选 `true`。
6. 验收预览、归档和报告后，再用 `dry_run=false` 发测试群。
7. 手工制作的“今日补充版”会在来源报告写入 `send_authorized=false`；只有完成确认后才能改为 `true` 并发送。
