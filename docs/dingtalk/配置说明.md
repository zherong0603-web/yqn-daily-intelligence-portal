# 配置说明

## 必填配置

### DINGTALK_WEBHOOK_URL

- 是否必须：真实发送必须；dry run 可不填。
- 打开哪里：GitHub 仓库 `Settings`。
- 点哪里：`Secrets and variables` -> `Actions` -> `New repository secret`。
- Name：`DINGTALK_WEBHOOK_URL`
- Value：钉钉测试群自定义机器人 webhook。
- 保存按钮：`Add secret`
- 配错后表现：发送步骤失败，或钉钉群收不到消息。

### DINGTALK_SECRET

- 是否必须：机器人启用加签时必须。
- 打开哪里：GitHub 仓库 `Settings`。
- 点哪里：`Secrets and variables` -> `Actions` -> `New repository secret`。
- Name：`DINGTALK_SECRET`
- Value：钉钉机器人加签 secret。
- 保存按钮：`Add secret`
- 配错后表现：钉钉返回加签校验失败。

## live 模式配置

如果你不想配置 OpenAI API，GitHub Actions 里的 live 模式会尝试使用 GitHub Models 免费限额。这个方案不需要手动填写 `OPENAI_API_KEY`，但受 GitHub 账号免费额度、接口稳定性和模型可用性限制；如果输出不符合 schema，系统会自动降级成安全 demo 日报。

### OPENAI_API_KEY

- 是否必须：OpenAI 原生 live 模式必须；GitHub Models 免费兜底不需要。
- 打开哪里：GitHub 仓库 `Settings`。
- 点哪里：`Secrets and variables` -> `Actions` -> `New repository secret`。
- Name：`OPENAI_API_KEY`
- Value：OpenAI API key。
- 保存按钮：`Add secret`
- 配错后表现：live 生成失败，demo 不受影响。

### OPENAI_MODEL

- 是否必须：OpenAI 原生 live 模式必须；GitHub Models 免费兜底不需要。
- 打开哪里：GitHub 仓库 `Settings`。
- 点哪里：`Secrets and variables` -> `Actions` -> `New repository secret`。
- Name：`OPENAI_MODEL`
- Value：账号可用的 OpenAI 模型名。
- 保存按钮：`Add secret`
- 配错后表现：live 生成失败或模型不可用。

## 手动运行

1. 打开 GitHub 仓库 `Actions`。
2. 选择 `DingTalk Morning Brief V1`。
3. 点 `Run workflow`。
4. `mode` 先填 `demo`。
5. `dry_run` 先保持 `true`。
6. 确认预览后，再改为 `false` 发测试群。
