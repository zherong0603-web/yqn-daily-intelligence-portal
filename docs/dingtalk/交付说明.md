# YQN 北美履约增长晨报 V1 交付说明

## 这是什么

这是每天自动发到钉钉测试群的《YQN 北美履约增长晨报》。它把公开政策、平台履约、北美仓配和小红书 B2B 线索经营信号，整理成团队当天能执行的动作。

## 每天几点发

工作日北京时间 8:45。系统使用 UTC 定时 `45 0 * * 1-5`，对应北京时间工作日 8:45。

## 发到哪里

V1 只发钉钉测试群。连续 3 个工作日稳定、内容不敏感、格式好看后，再考虑进入正式业务群。

## 群里会看到什么

- 今日一句话判断。
- 4 条核心信号：政策、平台、履约、增长。
- 每条信号包含发生了什么、为什么重要、YQN 可用点、今天动作、来源链接、置信度、是否敏感。
- 今日 3 个动作：销售、内容、履约或数据。
- 如果配置了网页地址，消息末尾会有归档链接。

## 哪些内容不会发

客户名单、客户联系方式、报价、合同、毛利、内部成本、未公开客户案例、销售聊天记录、私域客户明细、内部项目路径、API key、webhook、secret、个人副业、个人赚钱、用户个人叙事。

## 如何配置钉钉 webhook

1. 在钉钉测试群添加自定义机器人。
2. 复制机器人 webhook。
3. 打开 GitHub 仓库 `Settings`。
4. 进入 `Secrets and variables` -> `Actions`。
5. 点 `New repository secret`。
6. Name 填 `DINGTALK_WEBHOOK_URL`。
7. Value 粘贴钉钉测试群 webhook。
8. 点 `Add secret` 保存。

## 如何配置 DINGTALK_SECRET

如果钉钉机器人启用了加签：

1. 在机器人安全设置里复制加签 secret。
2. 打开 GitHub 仓库 `Settings`。
3. 进入 `Secrets and variables` -> `Actions`。
4. 点 `New repository secret`。
5. Name 填 `DINGTALK_SECRET`。
6. Value 粘贴加签 secret。
7. 点 `Add secret` 保存。

## 如何配置 OpenAI API

live 模式才需要：

1. 打开 OpenAI API key 页面，新建 API key。
2. 回到 GitHub 仓库 `Settings`。
3. 进入 `Secrets and variables` -> `Actions`。
4. 新建 `OPENAI_API_KEY`，Value 填 API key。
5. 新建 `OPENAI_MODEL`，Value 填账号可用模型。

demo 模式不需要 OpenAI API。

## 如何手动测试一次

1. 打开 GitHub 仓库 `Actions`。
2. 选择 `DingTalk Morning Brief V1`。
3. 点 `Run workflow`。
4. `mode` 选择 `demo`。
5. `dry_run` 保持 `true`。
6. 运行后下载 artifact 查看预览和 zip。

## 如何切换 demo/live

- `demo`：固定样例，适合验收格式，不依赖 OpenAI API。
- `live`：调用 OpenAI API 生成结构化日报，必须配置 `OPENAI_API_KEY` 和 `OPENAI_MODEL`。

## 如何避免误发正式群

- 先只配置钉钉测试群 webhook。
- 手动运行时默认 `dry_run=true`。
- 只有确认预览无误后，才把 `dry_run` 改成 `false`。
- 不要把正式业务群 webhook 填进 V1。

## 失败后看哪里

先看 GitHub Actions 失败步骤。常见原因：

- `OPENAI_API_KEY` 或 `OPENAI_MODEL` 缺失，live 模式无法生成。
- webhook 未配置，发送步骤会 warning 并跳过。
- 钉钉机器人加签配置错误，发送会失败。
- 内容命中风险词，会生成 `risk_report.json` 并阻断发送。

## 成本边界

demo 模式没有 OpenAI API 成本。live 模式按 OpenAI API 实际调用计费。钉钉机器人本身通常不产生 API 生成成本。

## 安全边界

GitHub Secrets 只保存密钥，不写进代码和网页。日志不打印 webhook、secret 或 API key。公开归档只放公开信息和可公开表达的业务动作。
