# Codex 卖家问题晨报影子运行规则

本阶段由 Codex / ChatGPT 自带的联网能力搜索和整理公开信息，不调用 OpenAI API，不使用 `OPENAI_API_KEY`，不进入钉钉发送链路。

核心原则：新闻不是晨报的主角，卖家正在发生的问题才是。新闻、平台公告和官方文件只负责证明问题真实存在、解释为什么现在值得关注。

## 每次必须完成

- 实际搜索美国、墨西哥、美墨联动三组信息。
- 先找卖家在公开社区、平台论坛和业务媒体中暴露的真实问题，再回到平台或政府原始页面核实事实。
- 默认搜索最近 24 小时；若不足以解释当前经营问题，可扩大到 7 天或引用仍在生效的规则，但必须说明为什么今天仍值得看。
- 固定产出 2 条美国、2 条墨西哥、1 条美墨联动。
- 每条同时保留两类证据：`seller_signal` 用来证明卖家正在讨论或遇到什么，`fact_basis` 用来证明规则、费用、时间和适用条件。
- 每个来源都保存核查时间；页面未公开发布日期时必须明确留空，禁止为通过校验猜测日期。规则有明确生效日时另外保存生效日。
- 政策、费用、平台规则和承运变化必须回到官网或平台原始页面；卖家帖子只能作为问题信号，不能单独证明普遍事实。
- 每条必须回答：卖家遇到什么、影响谁、为什么现在看、业务影响、市场如何使用、销售该追问什么、YQN 能承接什么。
- 固定结构校验、价值评分和来源校验通过后，仍需人工完成事实、业务价值、YQN 边界、群内可读性和发布设置五项验收。
- 卖家价值低于 70 分、缺少任一类证据、适用条件不清楚或配比错误时，验证失败。

## 发布时间与失败处理

- 06:00：Codex 开始联网采集和交叉核实。
- 07:15 前：生成结构化初稿。
- 07:30 前：完成自动校验，生成群预览和验收单。
- 08:15 前：验收人只判断事实、业务价值、YQN 边界和是否适合群内阅读。
- 08:35：冻结当天版本；冻结后不再临时替换核心结论。
- 08:45：正式发送窗口。前三个工作日只影子运行，不发送。
- 任一必查项失败、没有明确验收、目标群不匹配或重复检测异常时，当天停止发送并保留失败原因，不自动降级拼凑内容。
- 后续启用发送时，08:45 后只允许执行“确认未发送后的补发”，不得用多个定时任务盲目重复发送。

## 影子阶段安全边界

- 只允许写入 `data/codex-shadow/`。
- 每次输出 JSON、Markdown 预览和验证报告。
- 验证报告的 `send_allowed` 永远为 `false`。
- 不读取或更改 webhook、secret、API key 和钉钉开关。
- 没有足够强信号时不凑数，当次影子运行标记失败。
